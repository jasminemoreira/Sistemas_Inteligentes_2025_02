Tendência Central

Média, Mediana e Moda

O centro dos seus dados

As medidas de tendência central nos ajudam a encontrar o "valor típico" ou o centro de um conjunto de dados. Cada medida tem suas características específicas e é mais adequada para diferentes tipos de situações.

📊 Média Aritmética

x̄ = (x₁ + x₂ + ... + xₙ) / n
Soma de todos os valores dividida pelo número de observações

Quando usar: Dados numéricos sem outliers extremos.
Exemplo: Salário médio, nota média da turma.

📊 Mediana

Valor central quando dados estão ordenados
50% dos dados estão abaixo, 50% estão acima

Quando usar: Dados com outliers ou distribuições assimétricas.
Exemplo: Renda familiar, preços de imóveis.

📊 Moda

Valor que aparece com maior frequência
Pode ser única, múltipla ou não existir

Quando usar: Dados categóricos ou para identificar padrões.
Exemplo: Cor favorita, produto mais vendido.

Dica Prática para IA

Em machine learning, a escolha da medida de tendência central afeta o pré-processamento dos dados. Para imputação de valores faltantes, use a média para dados normais, mediana para dados com outliers, e moda para dados categóricos.

Dispersão

Variabilidade dos Dados

Quão espalhados estão seus dados?

As medidas de dispersão nos mostram quão espalhados ou concentrados os dados estão em relação ao centro. Essa informação é crucial para entender a qualidade e confiabilidade dos nossos dados.

📏 Amplitude

Amplitude = Valor Máximo - Valor Mínimo
Diferença entre o maior e menor valor

Medida simples, mas sensível a outliers. Útil para uma visão geral rápida da variação dos dados.

📐 Desvio Padrão

σ = √(Σ(xᵢ - x̄)² / n)
Raiz quadrada da variância

Medida mais robusta que considera todos os valores. Expressa na mesma unidade dos dados originais.

📊 Variância

σ² = Σ(xᵢ - x̄)² / n
Média dos quadrados dos desvios

Base matemática para muitos algoritmos de ML. Importante para análise de componentes principais (PCA).

📈 Coeficiente de Variação

CV = (σ / x̄) × 100%
Dispersão relativa em percentual

Permite comparar a variabilidade entre conjuntos de dados com diferentes unidades ou escalas.

Aplicação em IA: Normalização

O desvio padrão é fundamental na normalização Z-score: z = (x - μ) / σ. Esta técnica é essencial em redes neurais e algoritmos como k-means, pois garante que todas as variáveis tenham a mesma importância no modelo.

Distribuições

Padrões nos Dados

Como os dados se comportam

Compreender distribuições é fundamental para escolher algoritmos adequados, fazer suposições corretas e interpretar resultados em projetos de IA.

🔔 Distribuição Normal

Características: Simétrica, formato de sino, média = mediana = moda.

Regra 68-95-99.7: 68% dos dados estão a 1σ da média, 95% a 2σ, 99.7% a 3σ.

Aplicações: Muitos algoritmos assumem normalidade. Base para intervalos de confiança e testes de hipótese.

📊 Distribuição Uniforme

Características: Todos os valores têm a mesma probabilidade de ocorrer.

Exemplo: Resultado de um dado, números aleatórios em simulações.

Aplicações: Inicialização de pesos em redes neurais, amostragem aleatória.

📈 Distribuição Assimétrica

Assimetria à direita: Cauda longa à direita, média > mediana.

Assimetria à esquerda: Cauda longa à esquerda, média < mediana.

Exemplo: Renda populacional, tempo de resposta de sistemas.

📉 Distribuição Bimodal

Características: Dois picos distintos, pode indicar duas populações diferentes.

Exemplo: Alturas em uma população mista (homens e mulheres).

Aplicações: Clustering, segmentação de mercado.

Tipos de Dados

Classificação dos Dados

Diferentes tipos exigem diferentes abordagens

A classificação correta dos tipos de dados é fundamental para escolher as técnicas estatísticas e algoritmos de ML apropriados.

📊 Dados Quantitativos

Expressam quantidades numéricas

Discretos
  • Número de filhos
  • Quantidade de vendas
  • Número de cliques
  • Contagem de defeitos
Contínuos
  • Altura, peso
  • Temperatura
  • Tempo de resposta
  • Receita, lucro

📝 Dados Qualitativos

Expressam qualidades ou categorias

Nominais
  • Sexo (M/F)
  • Cor dos olhos
  • Marca do produto
  • Região geográfica
Ordinais
  • Escolaridade
  • Satisfação (1-5)
  • Tamanho (P/M/G)
  • Classe social

Impacto na Escolha de Algoritmos

Dados Quantitativos: Regressão linear, redes neurais, SVM.
Dados Nominais: Árvores de decisão, Naive Bayes, requerem encoding.
Dados Ordinais: Podem ser tratados como numéricos com cuidado, mantendo a ordem.

Estruturas

Organizando os Dados

Como estruturar dados para análise eficiente

A forma como organizamos e estruturamos nossos dados impacta diretamente na eficiência da análise e no desempenho dos algoritmos de IA.

📋 Dados Tabulares

Estrutura: Linhas (observações) × Colunas (variáveis)

Formato: CSV, Excel, bancos relacionais

Uso em IA: Algoritmos de ML tradicionais, análise exploratória

Exemplo: Dataset de vendas, informações de clientes

📊 Séries Temporais

Estrutura: Dados ordenados por tempo

Características: Tendência, sazonalidade, ruído

Uso em IA: LSTM, ARIMA, previsão de demanda

Exemplo: Preços de ações, vendas mensais, temperatura

🖼️ Dados Multidimensionais

Estrutura: Arrays n-dimensionais (tensores)

Formato: Imagens, vídeos, dados de sensores

Uso em IA: CNNs, deep learning, visão computacional

Exemplo: Imagens médicas, reconhecimento facial

📝 Dados Não-Estruturados

Estrutura: Texto livre, sem formato fixo

Características: Requer pré-processamento intensivo

Uso em IA: NLP, análise de sentimento, chatbots

Exemplo: Reviews, redes sociais, documentos

Princípios de Dados Limpos (Tidy Data)

1. Cada variável forma uma coluna
2. Cada observação forma uma linha
3. Cada tipo de unidade observacional forma uma tabela

Correlação

Relações entre Variáveis

Descobrindo conexões nos dados

Entender como as variáveis se relacionam é fundamental para feature engineering, seleção de variáveis e interpretação de modelos de IA.

📊 Correlação de Pearson

r = Σ(x-x̄)(y-ȳ) / √[Σ(x-x̄)²Σ(y-ȳ)²]
Mede relação linear (-1 ≤ r ≤ 1)

r > 0: Correlação positiva
r < 0: Correlação negativa
r ≈ 0: Sem correlação linear

🎯 Interpretação

|r| = 0.0 - 0.3: Correlação fraca

|r| = 0.3 - 0.7: Correlação moderada

|r| = 0.7 - 1.0: Correlação forte

Cuidado: Correlação ≠ Causalidade!

🔍 Correlação não-linear

Pearson pode não capturar relações não-lineares.

Alternativas:

• Spearman (monotônica)

• Kendall (baseada em ranks)

• Análise visual com scatter plots

Matriz de Correlação em IA

Ferramenta essencial para feature selection. Variáveis altamente correlacionadas podem causar multicolinearidade, prejudicando algoritmos como regressão linear. Use heatmaps para visualizar correlações de forma intuitiva.

⚠️ Armadilhas da Correlação

Spurious Correlation: Correlações espúrias podem aparecer por acaso ou por uma terceira variável oculta.
Exemplo: Vendas de sorvete e afogamentos estão correlacionados, mas a causa é a temperatura alta!

EDA

Análise Exploratória

Conhecendo seus dados antes da modelagem

A Análise Exploratória de Dados (EDA) é o primeiro e mais importante passo em qualquer projeto de IA. É aqui que conhecemos nossos dados e formamos hipóteses.

🔍 Inspeção Inicial

Shape dos dados: Quantas linhas e colunas?

Tipos de dados: Numéricos, categóricos, datas?

Valores faltantes: Onde e quantos?

Valores únicos: Cardinalidade das variáveis

📊 Estatísticas Descritivas

Para numéricas: Min, max, média, mediana, desvio

Para categóricas: Contagem de cada categoria

Distribuições: Histogramas e density plots

Outliers: Box plots e scatter plots

🔗 Relações Bivariadas

Numéricas vs Numéricas: Scatter plots, correlação

Categóricas vs Numéricas: Box plots por grupo

Categóricas vs Categóricas: Tabelas cruzadas

Target vs Features: Entender relações com variável objetivo

🎯 Insights e Hipóteses

Padrões encontrados: Documentar descobertas

Anomalias: Investigar valores suspeitos

Feature engineering: Ideias para novas variáveis

Estratégia de modelagem: Que algoritmos testar?

📋 Checklist de EDA

✓ Entendi o problema de negócio?
✓ Conheço a origem e significado de cada variável?
✓ Identifiquei padrões interessantes nos dados?
✓ Tratei valores faltantes e outliers adequadamente?
✓ Validei minhas descobertas com stakeholders?

Interpretação

Como Interpretar Resultados

Transformando números em insights acionáveis

A interpretação correta dos resultados estatísticos é fundamental para tomar decisões corretas e comunicar descobertas de forma eficaz.

📊 Contexto é Tudo

Significância Prática vs Estatística: Um resultado pode ser estatisticamente significativo mas praticamente irrelevante.

Domínio do Negócio: Conhecer o contexto ajuda a identificar resultados plausíveis ou suspeitos.

Limitações dos Dados: Sempre considerar vieses e limitações na coleta.

⚖️ Incerteza e Confiança

Intervalos de Confiança: Mais informativos que pontos únicos.

Margem de Erro: Comunicar a precisão das estimativas.

Poder Estatístico: Capacidade de detectar efeitos reais.

🎯 Comunicação Eficaz

Para Técnicos: Métodos, limitações, intervalos de confiança.

Para Gestores: Insights acionáveis, impacto no negócio.

Visualizações: Gráficos claros e auto-explicativos.

🚫 Erros Comuns

Cherry Picking: Selecionar apenas resultados favoráveis.

Overfitting nas Conclusões: Generalizar além dos dados.

Confundir Correlação com Causalidade: Associação ≠ Causa.

Template para Relatórios

1. Objetivo: O que queríamos descobrir?
2. Metodologia: Como analisamos os dados?
3. Resultados: O que encontramos? (com visualizações)
4. Limitações: O que pode afetar as conclusões?
5. Recomendações: Que ações tomar?

IA Ready

Preparando Dados para IA

Da estatística para machine learning

Os conceitos estatísticos que aprendemos são a base para preparar dados de qualidade para algoritmos de machine learning.

🔄 Normalização e Padronização

Min-Max Scaling: [0,1] - preserva distribuição original.

Z-Score: μ=0, σ=1 - assume distribuição normal.

Quando usar: Algoritmos sensíveis à escala (k-NN, SVM, redes neurais).

🎯 Tratamento de Outliers

IQR Method: Q1 - 1.5×IQR e Q3 + 1.5×IQR

Z-Score: |z| > 3 podem ser outliers

Ações: Remover, transformar ou manter dependendo do contexto.

🔢 Encoding Categórico

One-Hot: Para variáveis nominais

Label Encoding: Para variáveis ordinais

Target Encoding: Baseado na variável resposta

📊 Feature Engineering

Combinações: Criar novos features a partir dos existentes

Transformações: Log, sqrt para normalizar distribuições

Binning: Agrupar valores contínuos em categorias

Pipeline de Preparação

1. EDA completa → 2. Limpeza (missing, outliers) → 3. Feature engineering → 4. Encoding → 5. Scaling → 6. Split train/test → 7. Validação final
